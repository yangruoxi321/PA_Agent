## 1. 启动与初始配置

### 启动方式

```bash
# 方式一：直接运行
python run.py

# 方式二：模块方式运行
python -m pa_agent.main

# 方式三：安装后使用命令行入口
pip install -e ".[dev]"
pa-agent
```

### 首次运行

1. 程序启动后自动创建 `config/settings.json`（默认配置）
2. 如提示 MT5 未连接，确保 MetaTrader 5 终端已打开并登录
3. 首次使用需在 **菜单 → 设置** 中填写 API Key
4. API Key 通过 Windows DPAPI 加密存储，不会明文写入磁盘

### 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| Python | 3.11+ |
| MT5 终端 | 需运行并登录（用于获取实时行情） |
| 网络 | 需访问 AI API 服务（DeepSeek / PackyAPI 等） |

## 2. 主界面概览

程序主窗口分为**左侧图表区**和**右侧 AI 侧边栏**，顶部为操作控制栏。

```
┌──────────────────────────────────────────────────────────────────────┐
│ [品种▼] [周期▼] [K线数▼]  ☑等待收盘  [提交分析] [增量分析] [演示模式] │
├─────────────────────────────────┬────────────────────────────────────┤
│                                 │  ┌──────────────────────────────┐  │
│                                 │  │ 实时│决策树│可视化│决策│原始│调试│  │
│      K线图表（pyqtgraph）        │  ├──────────────────────────────┤  │
│      - 蜡烛图 + EMA20           │  │                              │  │
│      - 入场/止损/止盈线         │  │    AI 侧边栏内容             │  │
│      - 序号标签                 │  │    （按标签页切换）           │  │
│                                 │  │                              │  │
│                                 │  └──────────────────────────────┘  │
├─────────────────────────────────┴────────────────────────────────────┤
│ 状态栏：就绪 / 分析中 / 等待收盘…                                     │
└──────────────────────────────────────────────────────────────────────┘
```

### 控制栏元素

| 控件 | 功能 |
|------|------|
| **品种** | 可编辑下拉框，输入 MT5 品种名称（如 `XAUUSDm`、`US500m`） |
| **周期** | K线周期选择：1m / 5m / 15m / 1h / 4h / 1d |
| **K线数** | 抓取的K线数量（2–5000），默认 100 |
| **等待收盘** | 勾选后，提交分析时先等待当前未收盘K线走完 |
| **提交分析** | 发起完整两阶段 AI 分析 |
| **增量分析** | 基于上一次成功记录做增量分析（跳过已分析K线） |
| **演示模式** | 回放已保存的分析记录，无需真实 API 调用 |
| **图表实时更新** | 分析进行中图表暂停刷新，点击此按钮可恢复 |

## 3. 实时K线图表

### 图表特性

- **蜡烛图渲染**：使用 pyqtgraph，30Hz 刷新率
- **EMA20 叠加线**：自动计算并绘制 20 周期指数移动平均线
- **K线序号标签**：底部显示 K1（最新已收盘）、K2、K3… 逆向编号
- **交易决策叠加**：分析完成后自动绘制入场价、止损价、止盈价水平线
  - 入场线：白色实线
  - 止损线：红色虚线
  - 止盈线：绿色虚线
  - 方向标记：做多↑ / 做空↓

### 图表操作

- **鼠标滚轮**：缩放图表
- **鼠标拖拽**：平移图表
- 切换品种/周期时，图表自动重置

### 数据来源

- **主数据源**：MetaTrader 5（需 MT5 终端运行并登录）
- 默认 1 秒刷新间隔（可在设置中调整）
- 如果 MT5 未连接，状态栏显示"数据源未连接"提示

## 4. 提交分析（核心功能）

### 操作步骤

1. 在控制栏选择 **品种**、**周期**、**K线数**
2. 点击 **「提交分析」** 按钮
3. 图表自动冻结（暂停实时更新），状态栏显示"分析中…"
4. 右侧 AI 侧边栏切换到「实时」标签，展示分析流式输出
5. 分析完成后状态栏显示"分析完成"，图表恢复刷新

### 分析期间行为

- 图表冻结在提交瞬间的K线快照上
- 「图表实时更新」按钮变为可用，可手动恢复图表刷新
- 提交按钮锁定，不可重复提交
- 品种/周期切换会取消当前分析

### 注意事项

- 确保品种名与 MT5「市场报价」中名称完全一致（含后缀如 `m`）
- 品种名输入错误时，输入框下方显示红色警告
- K线数据不足时提示"数据不足，请等待缓冲区填满后再提交"

## 5. 增量分析

### 什么是增量分析

当同品种同周期已有成功分析记录，且新增已收盘K线数量不超过阈值（默认 10 根）时，提交分析自动走增量模式：

- AI 可看到上一次分析的结论作为上下文
- 只对新增K线做增量判断，而非从零开始
- 节省 token 消耗，提升分析速度

### 操作方式

- **自动增量**：当新增K线 ≤ 阈值时，点「提交分析」自动走增量模式
- **强制增量**：点「增量分析」按钮，无视阈值限制，强制基于上一次记录做增量
- 阈值在 设置 → 通用 → 「增量分析最大新增K线」 中配置（0 = 关闭自动增量）

### 增量分析失败的情况

- 未找到该品种/周期的成功分析记录
- 当前K线与上一轮记录无法对齐（缺口过大）
- 此时应改用「提交分析」做完整分析

## 6. 等待K线收盘后分析

### 使用场景

当前最新K线尚未收盘（正在形成中），希望等它收盘后再提交分析，确保数据完整性。

### 操作步骤

1. 勾选 **「等待最新K线收盘后再提交分析」** 复选框
2. 输入框旁出现倒计时（如"还剩 23 秒"）
3. 点击「提交分析」或「增量分析」
4. 程序等待当前K线收盘，倒计时实时更新
5. K线收盘后自动提交分析

### 取消等待

- 取消勾选复选框 → 取消等待
- 等待期间状态栏显示"等待当前K线收盘…还剩 XX 秒"

## 7. 两阶段分析流程详解

PA Agent 的核心分析采用**两阶段（Two-Stage）架构**：

### 阶段一：市场诊断（Stage 1 — Diagnosis）

**目的**：识别当前市场结构、通道形态与周期位置。

**流程**：
1. 组装阶段一 Prompt（K线数据 + 市场诊断框架 + 人设提示词 + 逐棒分析检查单 + K线信号识别 + 经验库案例）
2. 调用 AI API，流式输出
3. 解析返回的 JSON，验证格式
4. 提取关键字段：
   - `cycle_position`：市场周期位置（尖峰/微型通道/窄通道/正常通道/宽通道/趋势型交易区间/交易区间/极端交易区间）
   - `direction`：方向（bullish / bearish / neutral）
   - `gate_trace`：闸门决策路径
   - `gate_result`：闸门结果（proceed / wait / unknown）
   - `detected_patterns`：检测到的形态（楔形/反转尝试/H1H2/突破失败等）

### 阶段二：交易决策（Stage 2 — Decision）

**目的**：基于阶段一诊断结果，给出具体交易建议。

**流程**：
1. 根据阶段一的 `cycle_position` + `direction` + `detected_patterns`，路由加载对应策略文件
2. 加载经验库中相关案例
3. 组装阶段二 Prompt（策略文件 + 决策树 + 交易倾向 + 经验库 + 二元决策指引）
4. 调用 AI API，流式输出
5. 解析返回的 JSON，验证格式
6. 提取关键字段：
   - `order_type`：下单类型（限价单/突破单/市价单/不下单）
   - `order_direction`：方向（做多/做空）
   - `entry_price`：入场价
   - `stop_loss_price`：止损价
   - `take_profit_price`：止盈价
   - `trade_confidence`：信心分数（0–100）
   - `decision_trace`：决策树追踪路径
   - `terminal.outcome`：最终结果（trade / wait / reject）

### 闸门短路

如果阶段一的 `gate_result` 为 `wait` 或 `unknown`，阶段二将被跳过（闸门短路），直接输出"不下单"结论。

## 8. AI 侧边栏六大标签页

右侧 AI 侧边栏包含六个标签页，按功能分类：

### 8.1 「实时」标签

**主要功能**：展示 AI 分析的流式实时输出。

- 分析进行时，实时显示思考过程（reasoning）和正文内容（content）
- 阶段一/阶段二分别显示，有清晰的标题分隔
- 底部显示 **Token 用量进度条**：
  - 绿色（< 80%）：正常
  - 黄色（80%–95%）：警告
  - 红色（> 95%）：危险
- 分析完成后，底部出现 **自由对话输入框**，可与 AI 继续讨论
- 支持「重发上一条消息」（自动丢弃已发送消息中的思考内容以节省 token）

### 8.2 「决策树」标签

**主要功能**：以文本列表形式展示决策树追踪路径。

- 显示闸门（Gate）追踪：每个闸门节点的问题、回答、结果
- 显示策略（Decision）追踪：每个决策节点的问题、回答、结果
- 终端结果标注：wait / reject / trade / proceed

### 8.3 「决策树可视化」标签

**主要功能**：以科幻风格分支流程图可视化决策路径。

- 节点分为「闸门」和「策略」两种类型
- 分支路径用线条连接，动画播放从根节点到终端节点
- 自动播放功能：分析完成后自动播放路径动画
- 支持缩放和拖拽查看
- 颜色编码：
  - 绿色 → 是/trade
  - 红色 → 否/reject
  - 黄色 → 中性/wait
  - 蓝色 → proceed

### 8.4 「决策」标签

**主要功能**：以结构化卡片展示最终交易决策。

- **市场概览**：趋势方向、周期位置、市场阶段
- **交易决策**：下单类型、方向、入场价、止损价、止盈价
- **信心评分**：AI 信心分数（0–100），带颜色进度条
- **交易者方程**：风险回报比、预估胜率、是否通过交易者方程
- **诊断摘要**：市场结构诊断的简要说明

### 8.5 「原始」标签

**主要功能**：展示 AI 请求和响应的原始数据。

- 阶段一/阶段二的 System Prompt、User Prompt 完整内容
- AI 原始响应（含 reasoning_content）
- 验证信息：JSON 解析是否通过、缺失字段、无效字段
- 异常信息：网络错误、超时、JSON 截断等
- 「复制调试信息」按钮：一键复制完整调试数据

### 8.6 「调试」标签

**主要功能**：展示本次分析使用的所有提示词文件。

- **阶段一文件列表**：市场诊断框架、人设、K线信号识别等
- **阶段二文件列表**：根据路由加载的策略文件
- **经验库条目数**：本次分析加载的历史案例数量

## 10. 决策树与决策树可视化

### 决策树原理

PA Agent 内置一套**二元决策树**（基于 `prompt_engineering/二元决策.txt`），用于引导 AI 按固定逻辑链路做交易决策：

1. **闸门阶段（Gate）**：节点 0.1–0.3，判断市场是否具备交易条件
   - 0.1：是否有足够K线数据
   - 0.2：是否存在可识别的市场结构
   - 0.3：是否处于可交易环境（非铁丝网/极端混乱）
2. **策略阶段（Decision）**：节点 3–14，具体的交易逻辑决策
   - 入场信号识别（§9）
   - 入场方式选择（§11）
   - 止损止盈设定（§10.3）
   - 交易者方程验证
   - 硬性禁止项检查（§14）

### 可视化面板操作

- 分析完成后自动切换到此标签（如果设置了自动播放）
- 点击「播放路径」按钮手动播放
- 鼠标滚轮缩放，拖拽平移
- 设置中可配置默认缩放比例和播放时长

## 11. 分析后自由对话（Free Chat）

### 功能说明

两阶段分析完成后，可以在「实时」标签底部的输入框与 AI 继续对话：

- AI 已掌握本次分析的完整上下文（K线数据、诊断结果、决策过程）
- 每次发送消息时自动附加最新K线快照，确保 AI 看到最新市场数据
- 对话记录自动持久化到 `.followups.jsonl` 文件

### 使用方式

1. 分析完成后，输入框变为可用
2. 输入问题（如"如果明天跌破止损位怎么办？"）
3. AI 流式回复，实时显示
4. 可多次对话，上下文保持

### 注意事项

- 自由对话消耗额外 token
- Token 用量进度条实时更新
- 切换品种/周期会销毁当前对话会话

## 12. 演示模式

### 功能说明

使用已保存的分析记录（`records/pending/` 中的 `.json` 文件）回放界面，**无需真实 API 调用**。

### 启动方式

1. 点击「演示模式」按钮
2. 选择回放方式：
   - **手动选择记录**：弹出文件对话框，选择 `.json` 记录文件
   - **自动随机记录**：从 pending 目录随机选取记录

### 自动演示模式

- 自动模式下，每条记录回放完成后自动切换下一条
- 决策树可视化播放结束后自动切换到下一条记录
- 无可用记录时自动退出演示模式

### 退出演示

- 点击「退出演示模式」按钮（原「演示模式」按钮变为退出按钮）
- 提交真实分析时自动退出演示模式
- 切换品种/周期时自动退出

## 13. 设置对话框

通过 **菜单栏 → 设置 → 打开设置** 打开。

### AI 提供商设置

| 设置项 | 说明 | 默认值 |

| 模型 (model) | AI 模型名称 | `claude-sonnet-4-6` |

| Thinking | 是否启用扩展思考 | 开启 |

| Context Window | 上下文窗口大小 | 2,000,000 |
| 点击获取模型 API KEY | 打开帮助页面获取 API Key | — |

**常用 API 配置**：

| 提供商 | Base URL | 模型名 |
|--------|----------|--------|
| DeepSeek | `https://api.deepseek.com` | `deepseek-v4-pro` |
| PackyAPI | `https://www.packyapi.com/v1` | `claude-sonnet-4-6` |
| KKAI | `https://api.kkone.vip/v1` | — |
| 云雾 | `https://api.yunwu.ai/v1` | — |

### 通用设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 默认 Bar 数量 | 初始加载的K线数 | 100 |

| 决策树播放 | 分析完成后自动播放路径 | 开启 |
| 播放时长 | 路径动画持续时间 | 50 秒 |
| 决策树可视化默认缩放 | 相对整图适配的缩放百分比 | 500% |

## 14. 交易倾向（Decision Stance）

交易倾向影响**阶段二**的交易决策倾向，不影响阶段一的市场诊断。

### 四个倾向档位

| 档位 | 说明 | 典型行为 |
|------|------|----------|
| **保守** | 默认裁定标准，与旧版一致 | 优先清晰信号；边际情况倾向不下单；风险回报比要求 >= 1.5:1 |
| **均衡** | 比保守更愿意交易 | 允许次优但可执行的二类 setup；可接受 ~1.2:1 R:R；方向一致时优先考虑下单 |
| **激进** | 比均衡更愿意交易 | 接受更早更不完美的入场；可接受 ~1.0:1 R:R；主动寻找可下单方案 |
| **极度激进** | 强制选方向和进场方式 | **必须给出具体做多/做空方案**；不可因犹豫输出"不下单"（§14 硬性禁止除外） |

### 设置方式

1. 设置对话框 → 通用 → 交易倾向
2. 或直接修改 `config/settings.json` 中的 `general.decision_stance`

## 16. 数据源与K线刷新

### MT5 数据源

- 程序启动时自动连接本地 MetaTrader 5 终端
- 支持 MT5 所有时间周期：1m / 2m / 3m / 5m / 10m / 15m / 30m / 1h / 2h / 3h / 4h / 6h / 8h / 12h / 1d / 1w / 1M
- 品种名称需与 MT5「市场报价」中完全一致（含后缀如 `m`）

### 刷新循环（RefreshLoop）

- 默认每 1 秒从 MT5 拉取最新K线数据
- 新数据到达后自动更新图表
- 刷新间隔可在设置中调整（100–10000 ms）

### 数据缓冲区（KlineBuffer）

- 内存中维护最近 1000 根K线的环形缓冲区
- 切换品种/周期时自动清空并重新填充

### 图表冻结与恢复

- 提交分析时，图表自动冻结在分析瞬间的K线快照
- 防止分析期间K线更新导致 AI 看到的数据与图表不一致
- 点击「图表实时更新」可手动恢复

## 17. 经验库系统

### 目录结构

```
experience/
├── broad_channel/       # 宽通道
│   ├── success_cases/   # 成功案例
│   └── failure_cases/   # 失败案例
├── extreme_tr/          # 极端交易区间
├── micro_channel/       # 微型通道
├── normal_channel/      # 正常通道
├── spike/               # 尖峰
├── tight_channel/       # 窄通道
├── trading_range/       # 交易区间
├── trending_tr/         # 趋势型交易区间
└── unknown/             # 未知/其他
```

### 工作原理

1. 每次分析完成后，系统根据诊断结果的 `cycle_position` 在对应目录中加载经验案例
2. 成功案例和失败案例都会被加载
3. 经验案例作为 Prompt 的一部分发送给 AI，帮助其参考历史类似行情
4. 在「调试」标签中可查看本次加载的经验条目数

## 19. 安全与密钥管理

### API Key 加密

- 使用 Windows DPAPI（Data Protection API）加密
- 存储在 `settings.json` 中的 `api_key_encrypted` 字段（Base64 编码）
- 明文 `api_key` **永远不会写入磁盘**
- 加密/解密类：`pa_agent.security.secret_store.SecretStore`

### 日志脱敏

- 所有日志输出中 API Key 自动替换为 `****` 掩码
- 调试面板中 API Key 同样脱敏显示

### Git 安全

- `settings.json` 在 `.gitignore` 中排除
- 提供 `tools/setup_git_secrets.ps1` 脚本，设置 pre-commit hook 拦截敏感文件

## 20. 常见问题与故障排除

### 启动问题

| 问题 | 解决方案 |
|------|----------|
| `ModuleNotFoundError: No module named 'pa_agent'` | 在项目根目录执行 `pip install -e ".[dev]"` |
| `No module named 'PyQt6'` | 执行 `pip install PyQt6` |
| `DLL load failed` | 安装 Visual C++ 运行库 |
| MT5 未连接 | 确保 MT5 终端已打开并登录 |

### 分析问题

| 问题 | 原因与解决方案 |
|------|---------------|
| 品种红色警告 | 品种名与 MT5 不一致，检查后缀（如 `XAUUSDm`） |
| "数据不足" | 等待缓冲区填充（几秒后重试） |
| 阶段一/阶段二 JSON 验证失败 | 查看「原始」标签的验证信息；可能是 AI 输出被截断或格式错误 |
| JSON 截断 | 思考区 token 用尽导致正文未写完；可缩短 prompt 或调整 Reasoning Effort |
| 网络超时 | 检查网络连接和 API 服务可用性 |

### 配置问题

| 问题 | 解决方案 |
|------|----------|
| `settings.json` 损坏 | 删除该文件，程序自动重建默认配置 |
| API Key 无效 | 在设置中重新输入；点击"显示"确认输入正确 |
| 模型与 Base URL 填反 | 设置对话框会自动检测并提示 |

### 性能问题

| 问题 | 解决方案 |
|------|----------|
| 图表卡顿 | 减少 K线数量；降低刷新间隔 |
| Token 用量过高 | 减少 K线数量；使用增量分析；降低 Reasoning Effort |
| 分析耗时过长 | 使用更快模型；降低思考深度 |
