# PA_Agent 403 问题排查与解决

**时间**: 2026-06-10 23:13 - 23:40

## 问题
PA_Agent 通过 QClaw 网关调用 DeepSeek 模型时持续返回 403 (`code: 9002, auth_error, source: gateway`)，即使在 SPYDER IDE 控制台中直接用 OpenAI 库连接也报 403。

## 排查过程

1. **已确认正确的修复**：之前已修改 `deepseek_client.py`（扩展 `_is_deepseek_model` 判断）、`qclaw_connector.py`（默认 model 和 base_url）、`settings.json`（model 和 base_url）

2. **逐层验证**：
   - `exec` 工具中 `python` 直连 `19000/proxy/llm` → 200 OK ✅
   - 同一 Python 解释器、同一 openai 版本、同一 API key → `exec` 内 OK，Spyder 控制台 403
   - 无代理环境变量差异
   - `x-stainless-*` headers 不是原因
   - PA_Agent 的 DeepSeekClient 完整路径从 `exec` 运行 → 200 OK ✅
   - `sessions_spawn` 子代理也无法访问 19000

3. **根因**：QClaw 内部代理 `127.0.0.1:19000/proxy/llm` 仅允许 OpenClaw 自身进程树访问。Spyder 独立启动的 PA_Agent 不在该进程树内，被拒绝。

## 端点对比

| 端点 | 进程限制 | reasoning_content |
|------|---------|-------------------|
| `127.0.0.1:19000/proxy/llm` (内部代理) | 仅 OpenClaw 进程树 | ✅ 完整保留 |
| `127.0.0.1:65339/v1` (公开网关) | 无限制 | ❌ 被过滤 |

## 解决方案

**从 OpenClaw 进程树内启动 PA_Agent**，使其继承内部代理访问权限：

```powershell
Start-Process -FilePath "python" -ArgumentList "D:\cl\PA_Agent\run.py" -WorkingDirectory "D:\cl\PA_Agent"
```

最终配置 (`config/settings.json`):
```json
{
  "provider": {
    "model": "pool-deepseek-v4-pro",
    "base_url": "http://127.0.0.1:19000/proxy/llm",
    "api_key": "********",
    "thinking": true,
    "reasoning_effort": "max"
  }
}
```

## 修改文件
- `pa_agent/ai/deepseek_client.py`：新增 `_is_deepseek_model()` 函数，扩展 thinking 参数判断
- `pa_agent/ai/qclaw_connector.py`：默认 model 改为 `pool-deepseek-v4-pro`，base_url 改为 19000
- `config/settings.json`：model、base_url 修正

调试代码已清理。
