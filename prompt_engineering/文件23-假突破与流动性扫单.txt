【文件定位】本文件职责：处理「假突破 / 流动性扫单（failed breakout / liquidity grab）」——一个只在结构边沿触发的 setup，含两个镜像方向：
- **下沿做多（failed_breakout_below）**：向下刺穿已识别支撑一点点后收回（下破假突破/下沿扫单）→ 边界限价做多。
- **上沿做空（failed_breakout_above）**：向上刺穿已识别阻力一点点后收回（冲高假突破/上沿诱多扫单）→ 边界限价做空。

用于阶段二在 detected_patterns 含 failed_breakout_below / liquidity_grab_candidate / failed_breakout_above / liquidity_grab_above_candidate / *_pending 时，决定是否在边沿提前挂限价单并设保护止损。本文件不生成战略 thesis，只在价格结构层面处置已识别的扫单候选。

## 0. 铁律（不可妥协，违反即不下单）

- 该判定**只在 K 线收盘后**做，**永远不在刺穿进行中**做。绝不对未收盘的 bar 标「扫单」。
- 绝不在「正在刺穿」的过程中产出任何反转信号。实时刺穿与真破在反转发生前长得完全一样，**不准假装能实时区分**。
- 一个「扫单候选」至少需要：刺穿的那根 bar 已收盘，且有**反方向的下一根跟随**确认（下沿要向上跟随，上沿要向下跟随）。
- 程序侧已有确定性守卫（liquidity_grab 字段，含 side=below/above、direction=long/short）：
  - `status=candidate`：六条几何 + 边沿位置全部通过，可按本文件处置。
  - `status=pending`：最近一根是新鲜刺穿、尚无收盘跟随棒 → **只能观察、提前挂单，不得据此市价下单**。
  - `status=downgraded` / 无 liquidity_grab 字段：降级为普通突破/普通走势，不产出本 setup。
- 若 stage1 提供 `liquidity_grab.confidence_cap`，本 setup 的 trade_confidence **不得超过该上限**。

## 1. 这是什么

流动性扫单 = 价格短暂刺穿一个**已识别的边界结构**，触发该位之外的止损/突破挂单池（止损爆发），随即被反向盘吸收、收回结构内。
- 下沿：刺穿支撑 → 触发空头止损 + 抄底诱空，下破失败 → 做空者被套（trapped bears）→ 倾向回区间上沿。
- 上沿：刺穿阻力 → 触发多头追突破 + 诱多，冲高失败 → 做多者被套（trapped bulls，常伴买进高潮/climax）→ 倾向回区间下沿。

它**不是**：随便一根影线、区间中部的反转、与刺穿方向一致的 Always-In 背景下的突破（那更可能是真破）。

## 2. 判定为「扫单候选」须同时满足（缺一即降级）

① **刺穿已识别边界**：刺穿的是 support_levels / resistance_levels 里已有的结构（前低/前高/缺口边/区间边沿/关键均线），且**只刺穿一点点**。不准用随机影线硬凑。
② **突破失败**：刺穿后在同根或随后 1–2 根内**收回结构内**（下沿收回支撑上方 / 上沿收回阻力下方）。
③ **K线形态**：长影 + 实体小 + 收在反转一侧。下沿=长**下**影、收上半部（理想上 1/3）；上沿=长**上**影、收下半部（理想下 1/3）。
④ **量**：刺穿放量（止损/突破爆发）后量缩/被吸收。**缺量降级**：tick volume（外汇/黄金/MT5）或无量时，本条仅供参考，**trade_confidence 封顶下调**并标注「成交量为 tick/缺失，条件④未确认」。
⑤ **背景**：trading_range 或反向结构，**不与刺穿同向的 Always-In**。下沿禁 Always-In-Short；上沿禁 Always-In-Long。
⑥ **反向跟随**：下一根有反方向跟随确认（下沿向上、上沿向下）。
⑦ **边沿位置**（§6.3/§14 硬约束，程序强制）：被刺穿结构必须位于窗口区间的**外沿**——下沿候选在下 1/3、上沿候选在上 1/3，**绝不在中间 1/3**。程序按 `(结构位 - 窗口最低)/(窗口最高 - 窗口最低)` 判定（下沿 ≤0.40，上沿 ≥0.60）。

## 3. 触发后的处置——边界限价单 + 止损，不是市价追

确认候选后，**不要市价追**。按三价方案产出（order_type=限价单）：

**下沿做多（side=below, direction=long）**
- entry：挂在**被刺穿支撑下方一点**（止损池位置，Al Brooks 边界限价单）。
- stop：放在**下影低点之下**。继续跌（真破）则被带出场。
- target：回**区间上沿 / 前高**。
- order_direction=做多。

**上沿做空（side=above, direction=short）**
- entry：挂在**被刺穿阻力上方一点**（突破追单/止损池位置）。
- stop：放在**上影高点之上**。继续涨（真破）则被带出场。
- target：回**区间下沿 / 前低**。
- order_direction=做空。

两者共同：**invalidation_condition** = 收盘越过影线极点 = 结构失效（真破），止损生效。输出里必须显式注明：**「即使全部条件满足，也只是概率更高、并非保证；真正的保护是 stop，不是这个识别。」**

## 4. 与基本面的耦合（基本面只能降级，不能凭空发起）

- fund flows / analyst / short interest 一律**只作背景**，不发起本信号。
- 若刺穿伴随**反向真实催化裂缝**（程序侧：context_assessment.stance=diverges 且方向相反——下沿做多遇 confidence_adjustment ≤ -10 的看空催化、上沿做空遇 ≥ +10 的看多催化）→ **调低或否决**：裂缝背景下的突破更可能是真破。程序守卫已会在此情形 downgrade。
- 若只是噪音 / 无催化 → 不干预价格层面判定。

## 5. 与 §6.3 / §14 的强制一致

- 本 setup 是**边沿** setup：只在结构外沿触发，**绝不在区间中部（中间 1/3）给信号**（§6.3「中间 1/3 不交易」、§14「禁止区间中部入场」）。
- 与文件18（突破失败与突破测试）、文件22（信号失败后的磁力位）一致：被扫的止损/挂单池 = 磁力位；收回后区间对侧为目标磁力位。
- 上沿诱多扫单常与买进高潮（climax_triggered）、被套多头（trapped_traders）、突破失败（breakout_failure）同现；下沿扫单常与空头被套同现。

## 6. 降级清单（任一成立 → 不产出本 setup，按普通走势处理）

- 刺穿过深 / 收盘明显留在结构外（真破，非扫单）。
- 无长影 / 大实体 / 收在反方向半部。
- 背景为同向 Always-In 或强势同向尖峰。
- 无反向跟随（status=pending 时只挂单观察，不确认）。
- 反向出现真实催化裂缝。
- 刺穿的不是已识别边界（无 support_levels / resistance_levels 依据）。
- 结构不在外沿、落在中间 1/3。
