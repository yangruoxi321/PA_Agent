## 一、Spike 定义与路由的级联矛盾

### 1.1 Spike 最低根数与追单门槛不匹配

| 来源 | 定义 |
|------|------|
| `极速上涨分析识别.txt` §三 | 2根=最低可路由尖峰，"可输出 spike 路由" |
| `二元决策.txt` §3.1 | 2根=最低可路由尖峰，同上 |
| `极速上涨交易策略.txt` §三 | 追单条件=**3+根**大型趋势棒 |
| `二元决策.txt` §3.3 | 追单=**3+根**强趋势棒 |

**矛盾**：2根已确认为spike并路由至spike策略文件，但spike策略文件中追单需要3根。那么2根spike被路由后唯一能做什么？文件没有给出2根spike的具体行动指令——既不能追单，策略文件也未定义"2根spike的专属操作"。**2根spike被创建了一个策略路由，但该路由下无可用操作**。

**建议**：明确2根spike的可用行动空间——例如"2根spike=确认spike路由但仅允许等待回撤入场(SPS)，禁止SCS追单"。或在二元决策§3.3中增设2根条件的处理分支。

### 1.2 1根"弱尖峰候选"的路由模糊

`极速上涨分析识别.txt` 和 `二元决策.txt` 均定义1根为"弱尖峰候选"，spike_stage=active（弱），"禁止追单，等第二棒确认再评估"。

**问题**：1根"弱尖峰候选"时，cycle_position 该输出什么？
- 如果输出 `spike` → 策略路由会加载 spike 策略文件 → 但 spike 策略文件没有1根的操作（SCS需要3根，SPS需要回撤，FFES需要旗形）
- 如果输出 `trend` / `channel` → 就失去了"等第二棒确认"这个最重要的指令
- 如果输出 `unknown` → 会走到未知态不加载任何策略

当前没有任何地方明确1根候选的 cycle_position 值。

**建议**：明确1根弱候选的输出枚举值，并在策略加载侧有对应的处理逻辑（wait + watch_points 写清第二棒确认条件）。

### 1.3 尖峰结束的硬性规则 vs 高潮判定的软性规则

| 来源 | 规则 |
|------|------|
| `极速上涨分析识别.txt` | **硬性规则**：第一条暂停棒或回撤棒出现=尖峰立即结束 |
| `二元决策.txt` §3.6 | 高潮=任意序列+衰竭信号（长尾线/小实体/反向棒）触发 |
| `市场诊断框架.txt` 状态1 | 同上，以衰竭信号为准 |

**矛盾**：
- 6根趋势棒 + 第7根是小实体（但不是回撤棒，收盘仍创新高）→ 按硬性规则尖峰未结束（没有暂停/回撤棒），但按高潮规则6根进入预警+小实体触发高潮 → 尖峰还在？高潮已触发？
- 7根趋势棒 + 第8根是暂停棒 → 按硬性规则尖峰结束，但8根未出现衰竭信号 → 是高潮吗？

**根本问题**：尖峰的"结束"（结构边界）和高潮的"触发"（风险事件）是两个独立维度，但被放在了同一决策节点上。

**建议**：解耦这两个维度——尖峰结束是结构判断（影响后续路由：→通道/区间/反转），高潮是风险判断（影响仓位和入场禁令），不应混用同一个"是否出现高潮"的二元问题。

## 二、止损规则的多源冲突（核心矛盾）

### 2.1 信号棒极点 vs 尖峰回撤极点

| 来源 | 止损基准 |
|------|---------|
| `二元决策.txt` §10.1 | 统一止损=信号棒极点外1跳 |
| `文件17` §一.2.a | 同上，唯一优先方法 |
| `文件16` §七.1 (SCS) | "最近一根回撤K线的极点外1跳动（若无回撤，则最近一根趋势棒的低/高点外1跳动）" |
| `极速上涨交易策略.txt` §一 | 尖峰期间止损=尖峰起点下方 |

**矛盾**：尖峰追单(SCS)没有传统意义上的"信号棒"——是连续趋势棒的直接追入。SCS的止损用了"最近回撤K线极点"或"最近趋势棒极点"，这与"信号棒极点外1跳"的通用规则完全是两套逻辑。

**影响**：当二元决策§10.1统一执行"止损=信号棒极点外1跳"时，尖峰场景下无法找到"信号棒"。

**建议**：在§10.1的统一止损表中增加尖峰专用行，明确SCS/SPS/FFES各自的止损基准，避免通用规则被错误应用到尖峰场景。

### 2.2 缓冲跳动数不统一

| 来源 | 缓冲量 |
|------|--------|
| `二元决策.txt` §10.1 通用 | 1跳 |
| `二元决策.txt` §10.1 宽通道 | "波段极点外1跳"（实际可能2-3跳） |
| `二元决策.txt` §10.1 铁丝网 | "至少3-5跳缓冲" |
| `极速上涨交易策略.txt` §七 | "至少留出1-2个跳动点的缓冲" |
| `震荡区间交易策略.txt` §二 | "信号棒低点/高点外1跳动，边界作为结构参考" |

**矛盾**：同样是在区间边界做反转刮头皮，`震荡区间交易策略` 说1跳，铁丝网规则说3-5跳。边界什么时候算"铁丝网"，什么时候算"普通区间边界"？临界点是30%的紧凑度，但紧凑度32%的区间=1跳，28%=3-5跳？这是不合理的阶跃。

**建议**：缓冲跳动数应为连续函数或至少给出过渡区间的处理规则（如紧凑度25%-35%用2跳过渡）。

### 2.3 "资金管理止损"的角色分裂

`文件17` 和 `二元决策.txt` §10.2 都强调：8跳/信号棒60%高度是"风险过滤器"而非止损位置。

但 `文件17` §一.4 信号棒过长处理方案中写着"用资金管理止损作为风险上限参考：固定8跳动止损"。虽然加了"参考"二字，但在实战中agent很容易理解为"信号棒太长=改用8跳止损"。

**矛盾**：如果信号棒过长导致结构止损>8跳，规则说"放弃或等更小信号"。但如果市场没有给出更小信号呢？agent只能无限等待——而此时资金管理止损恰好提供了一个"可行的止损位置"，产生了强迫性的二选一场景。规则应明确：**信号棒过长+无替代信号=放弃本信号周期，而非降级到固定跳数止损**。

## 三、盈亏比上限与结构目标的冲突

### 3.1 1.5R上限 vs 161.8%测量运动

| 来源 | 盈亏比规则 |
|------|-----------|
| `二元决策.txt` §10.3 | **硬上限**："盈亏比上限 1.5:1" |
| `文件17` §二.1.b | "PA Agent 的盈亏比上限"=1.5R |
| `极速上涨交易策略.txt` §七 | 目标类型="100%等距、138.2%/161.8%测量运动" |

**矛盾**：161.8%测量运动=1.618R，直接超过1.5R上限。138.2%=1.382R，在边界上（含滑点则可能超过）。

**具体场景**：尖峰高度30点 → 100%目标=30点盈利，若止损10点=3R（远超1.5R）。即使用50%仓位，按1.5R上限也只能预期15点盈利，与任何测量运动目标都不匹配。

**建议**：为尖峰场景设专项盈亏比规则——尖峰的测量运动目标具有结构支撑，可以突破通用1.5R上限。或者统一：尖峰交易目标=min(测量运动目标, 1.5R止损距离)，明示取舍。

### 3.2 分批出场与1.5R的关系模糊

`文件17` §二.3：分批出场"第一批在第一个结构目标（1R~1.5R），第二批在第二个结构目标"

**问题**：第一批已到1.5R，剩余仓位用跟踪止损。但如果第一批目标本身就是1.5R，而且是通过交易者方程的底线——那剩余仓位实际上没有数学保障（因为整体盈亏比已经触顶1.5R，剩余仓位只是"免费博"）。规则应明确剩余仓位是零风险博取还是也需要独立方程检验。

## 四、通道分类维度的不一致

### 4.1 双维度分类的交叉盲区

| 分类维度 | 窄通道 | 宽通道 | 来源 |
|---------|--------|--------|------|
| 斜率 | >45° | <30° | `通道宽度判断.txt`（from summary） |
| 回撤幅度 | 极小 | 50%-78.6% | `二元决策.txt` §5.2/§5.4 |
| 回撤幅度 | 20-40% | — | `上涨通道分析识别.txt`（from summary） |

**矛盾**：斜率45°和回撤幅度不是一一对应关系。存在：
- 低斜率(<30°)但回撤小(<30%)的市场 → 宽通道还是窄通道？
- 高斜率(>45°)但回撤大(>50%)的市场 → 发生了反转？还是宽通道？

当前决策树只按回撤幅度走（§5.2→§5.3→§5.4），完全没有使用斜率判断。但如果agent在诊断时用了`通道宽度判断.txt`的斜率标准，就会与二元决策产生分歧。

**建议**：明确通道分类的主维度。建议用回撤%作为路由主键（与二元决策对齐），斜率作为辅助描述。或将30°-45°斜率+30-50%回撤的市场定义为"常规通道"。

### 4.2 "常规通道"定义缺失量化

`二元决策.txt` §5.3 常规通道："回撤30%-50%，波段清晰，通道斜率适中"——"适中"没有量化。

而 §5.2 窄通道定义为"斜率大，回撤小"也没有量化。

**问题**：Agent 如何区分"斜率大"和"斜率适中"？没有量化标准则每次判断都会不一致。

## 五、三角形形态的存在性矛盾

| 文件 | 三角形处理 |
|------|-----------|
| `震荡区间分析识别.txt` §五 | 详细定义了4种三角形（上升/下降/扩张/对称），含交易规则 |
| `震荡区间交易策略.txt` §四 | 三角形交易规则（4种，具体流程） |
| `二元决策.txt` | **完全没有三角形**的决策分支 |
| `pattern_enum.md` | **不包含任何三角形相关枚举值** |

**矛盾**：震荡区间文件中有完整的三角形定义和交易规则，但决策树(二元决策.txt)中没有三角形分支，模式枚举(pattern_enum.md)中也没有三角形标签。

**影响**：即使 Agent 正确识别了三角形形态，也找不到路由目标。无法在 `detected_patterns` 中写入合法枚举值。这是一个**完整功能缺口**——三角形分析存在但无法接入决策流。

**建议**：
1. `pattern_enum.md` 增加 `ascending_triangle`、`descending_triangle`、`symmetrical_triangle`、`expanding_triangle` 枚举
2. `二元决策.txt` 增加 §6.6 三角形分支或将其作为 §6 内的子状态
3. 明确三角形的 breakout_pullback 入场与现有突破测试规则的关系（三角形突破=突破测试入场的高概率版本）

## 六、区间密度的双指标体系冲突

| 指标 | 定义 | 阈值 | 来源 |
|------|------|------|------|
| 紧凑度 | 区间宽度/平均波段幅度 | <30%=Barbwire | `震荡区间分析识别.txt` §二 |
| 变异系数(CV) | 波段幅度标准差/均值 | CV<0.3=高密度 | `震荡区间分析识别.txt` §三 |

**矛盾**：这是两个不同维度的指标：
- 紧凑度衡量的是区间宽度相对于外部波段的窄度
- CV衡量的是区间内部波段的一致性

一个区间可以同时是"高CV（内部波段不一致）+ 低紧凑度（整体窄）"——这时应该按哪个指标决策？
- 紧凑度<30% → Barbwire → 不交易
- CV<0.3 → 高密度 → 推荐交易

**具体陷阱**：一个区间宽度是平均波段的25%（Barbwire条件满足），但内部10个波段中有5个10点、5个20点（CV约0.4, 中等密度）。按Barbwire规则不交易，按密度规则可交易（密度中等）。两个规则给出了相反的答案。

**建议**：明确指标的优先级——Barbwire（紧凑度）优先于密度（CV）。或单指标化：将紧凑度和CV合并为一个综合质量评分。

## 七、突破交易规则的三段不连续

### 7.1 突破本身 → 突破测试 → 区间变趋势，三段的衔接断层

| 阶段 | 规则 | 来源 |
|------|------|------|
| 突破中 | "3-5根强趋势棒+实体大影线短=高质量真突破" | `二元决策.txt` §6.5 |
| 突破后 | "不追第一根突破...等回踩测试...测试成功后顺势入场" | `二元决策.txt` §6.5 |
| 趋势确认后 | "连续3-5根强趋势棒突破边界+有跟随+回踩测试成功+EMA倾斜"→切换趋势策略 | `二元决策.txt` §13.4 |

**矛盾**：三段用了几乎相同的条件（3-5根强趋势棒），但得出了三个不同的行动指令：
1. §6.5 第一段："这是真突破"（但等测试）
2. §6.5 第二段："不追"（等测试成功）
3. §13.4："切换为顺势回撤入场"（此时可以入场了）

Agent 在"突破中"看到3-5根趋势棒→判断为"真突破"→但被告知"不追"→那真突破的判断有什么作用？真突破和假突破的区别只在于后续有没有回踩测试成功，而这是事后才知的。

**建议**：突破中不问"真/假"，只问"质量"。"真突破"是事后标签，不是实时决策依据。将§6.5的"高质量真突破"改为"高质量突破"，移除真/假判断，只保留质量评级+等测试的统一指令。

## 八、缺口棒定义的双重语义

| 来源 | 定义 |
|------|------|
| `文件16` §五.1 | "缺口棒特指均线缺口棒(Moving Average Gap Bar)，不是开盘跳空缺口" |
| `极速上涨分析识别.txt` §一 四位一体 | "缺口：从开盘角度描述——大幅跳空高开" |

**矛盾**：同一系统内"缺口"有两个完全不同含义——一个是均线偏离（技术指标概念），一个是开盘跳空（价格行为概念）。新Agent或新用户会混淆。

**建议**：统一术语——均线缺口棒保持"均线缺口棒/MA Gap Bar"，开盘跳空使用"跳空缺口/Opening Gap"。

## 九、策略加载的架构冲突

### 9.1 诊断与策略的循环依赖

| 文件 | 声明 |
|------|------|
| `市场诊断框架.txt` | "阶段一不制定具体下单、止损、止盈方案" |
| `震荡区间分析识别.txt` | "读取前必须先完成市场诊断框架的阶段一" |
| `极速上涨分析识别.txt` | 同上 |
| `二元决策.txt` | 包含§0-§2诊断 + §3-§14策略，是完整两阶段 |

**矛盾**：
- 市场诊断框架说它是阶段一，二元决策是阶段一+阶段二
- 但具体的分析识别文件（震荡区间/极速上涨/通道）又说它们需要在市场诊断框架之后加载
- 二元决策的§1-§2与市场诊断框架的"三层递进"有大量重叠

**实际加载顺序不明确**：
1. 加载市场诊断框架（三层递进）→ 确定cycle_position
2. 加载二元决策 → 又做一遍诊断（§1-§2）
3. 加载具体的分析识别文件 → 再做第三遍诊断

三步诊断有大量冗余，且如果三步得出不同结论则无法仲裁。

**建议**：明确唯一的诊断入口。建议方案：
- 市场诊断框架 = 唯一诊断层（阶段一）
- 二元决策的§1-§2 = 移除，替换为"参见市场诊断框架输出"
- 具体策略文件（分析识别+交易策略合并？） = 阶段二

### 9.2 文件职责边界模糊

"分析识别"文件和"交易策略"文件被分成了独立文件（如`上涨通道分析识别.txt` + `上涨通道交易策略.txt`），但：
- 分析识别文件中也含有"交易策略"内容（如`震荡区间分析识别.txt` §二 Barbwire 交易策略）
- 交易策略文件中也含有"识别"内容（如涨跌通道交易方案中的通道类型区分）
- `二元决策.txt` 中§4-§8又包含了通道/区间/尖峰的识别条件

同一个"这是通道吗？"的判断在三个不同的地方有三种不同详细程度的规则。

**建议**：识别和策略严格分离，或干脆合并。当前半分离状态增加了维护成本且容易引入不一致。

## 十、二次入场vs二次高潮的语义冲突

### 10.1 "第二次"的两套逻辑

| 上下文 | "第二次"的含义 | 规则 |
|--------|---------------|------|
| `二元决策.txt` §3.7 | 第二次高潮 | 不追，保护利润 |
| `二元决策.txt` §7.3 | 二次入场（反转） | 胜率更高，优先考虑 |
| `二元决策.txt` §9.4/§9.6 | 第二次入场（通用） | 胜率更高 |
| `文件16` §五.2 20GB | 第二次均线缺口棒尝试 | 优先等待，失败后不做第三次 |
| `二元决策.txt` §7.4 | 二次入场失败 | 不做第三次 |

**矛盾**：同样是"第二次"：
- 反转的第二次入场→优先
- 高潮的第二次→危险→不追
- 20GB的第二次→优先等待
- 但第二次入场失败→禁止第三次

Agent 需要区分"这是高潮的第二次还是反转入场的第二次"——两者可以同时发生（反转发生在高潮后）。此时"第二次"是利好还是利空？

**建议**：为"第二次"增加明确的上下文前缀——second_climax、second_entry、second_gap_test——避免裸用"第二次"造成歧义。

## 十一、补充矛盾（快速列举）

### 11.1 20GB 在决策树中缺失

`文件16` 有完整的20GB定义和交易规则，但 `二元决策.txt` 完全未提及20GB。Agent 即使检测到20GB也不知道在决策树的哪个节点处理。

### 11.2 Always In 与反转的硬锁死

`二元决策.txt` §2.4 + §14：AIL中不做首个做空反转信号，AIS中不做首个做多反转信号。只有在"趋势线突破+前极点测试失败"后才可进入反转。

**问题**：如果在AIL中出现了非常强的双棒反转+外包棒+均线缺口，但尚未满足"趋势线突破+前极点测试失败"的硬条件，Agent 只能坐视不理。这个锁死程度是否过于刚性？

### 11.3 止损移动禁令 vs 持仓管理的止损收紧

| 来源 | 规则 |
|------|------|
| `二元决策.txt` §12.3 | "不随意移动止损" |
| `震荡区间交易策略.txt` | "不要移动止损" |
| `文件17` §一.4 | 入场棒强势→收紧止损至入场棒极点 |
| `二元决策.txt` §12.1 | "止损可收紧至入场棒极点" |

**矛盾**：止损收紧到入场棒极点=移动止损。到底是"不移动"还是"只能单向收紧"？文件17和二元决策§12.1允许收紧，但区间文件说不移动——这是针对不同场景的不同规则还是矛盾？

### 11.4 pattern_enum 与 detected_patterns 的覆盖缺口

`pattern_enum.md` 明确说 "detected_patterns 只放结构候选"。但以下在策略文件中被详细定义的结构形态不在枚举中：
- 三角形（4种）
- 抛物线楔形/抛物线通道
- 收缩台阶/shrinking_stairs（枚举中存在但未被二元决策的§5.5完全覆盖）
- 买进高潮/climax → 枚举中存在 ✓
- Micro Channel → 存在 ✓
